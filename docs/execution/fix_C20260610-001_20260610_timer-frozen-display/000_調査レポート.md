## 1. 症状
活動セットの「開始」を押して計時中（running）になっても、画面の「経過時間」表示が `00:00` のまま固まり、増えていかない。アイテムを「終了」すると、記録された時間は実際の経過分だけ進んでいる。

## 2. 期待挙動
- (A) 計時中は経過時間がリアルタイムにカウントアップする（壁時計の経過に追従）。
- (B) 計時中のアイテムは経過時間に加え、開始時刻・現在時刻も併せて表示される（経過時間だけでは把握しづらいため）。

## 3. 再現手順
1. 任意の活動セットを開く。
2. 「開始」ボタンを押す（status=running）。
3. 数秒〜数十秒待ちながら「経過時間」表示を観察する。
   → `00:00` のまま変化しない。
4. 「終了」または「セット終了」を押す。
   → 確定後の記録には実経過秒が入っている（時間は進んでいた）。

## 11. タイムラインから抽出した仮説
- 仮説 1: 表示が「保存済み `elapsedSec`（計時中は初期値 0）」を読んでおり、計時中に `now() - startedAt` を導出していない（SPEC の経過モデルと表示の乖離）。
- 仮説 2: 計時中に再描画する tick（interval 等）が存在せず、状態遷移時以外は再レンダリングされない。
