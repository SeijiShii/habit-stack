# execution E2E テスト計画（活動を 1:N period モデルにして中断時間を経過から除外）

## 1. 変更 UC シナリオ

### UC: 中断時間が経過に計上されない（claim C20260614-001 の是正）
| シナリオ ID | 前提 | 操作ステップ | 期待結果 |
|---|---|---|---|
| E1 | 複数アイテムのセット | 活動 A 開始 → しばらく計時 → 一時停止 → （**同じ活動を再開せず**）「次の活動へ」 | 活動 A の確定経過 = 計時していた区間のみ。中断していた区間は**含まれない**（セット合計が中断分だけ膨らまない） |
| E2 | 計時中 | 一時停止 → 同じ活動を再開 → 計時 → もう一度 一時停止 → 再開（多重中断） | 経過は計時区間の合計のみ。中断 2 回分が両方とも除外される |
| E3 | 計時中 | 一時停止 → 再開せず「セット終了」 | 完了。活動 A の経過に中断区間が含まれない |
| E4 | 計時画面表示 | 中断・再開を複数回行った活動を表示 | 開始時刻表示は**最初の開始時刻のみ**（periods を全部表示しない） |

## 2. リグレッションシナリオ（既存 UC、重要度高）
| UC | シナリオ ID | 確認観点 |
|---|---|---|
| 通常計時（pause なし） | R1 | 一度も中断しない活動の経過が従来どおり正確（mm:ss・開始/現在時刻表示が不変） |
| 4H 上限クランプ | R2 | 長時間（口が開いたまま）放置で 1 活動 4H に上限クランプされる |
| 永続/復帰（R20260611-001） | R3 | リロード/復帰後も periods が復元され、経過・セット合計が正しく復元される |
| 達成記録 | R4 | 実行確定で達成日が記録される（穴あき許容、≥1 で達成） |
| セット合計（R20260613-004） | R5 | 計時画面のセット合計表示が periods 経路でも正しく累積する |
| 振り返り総覧 | R6 | 振り返り側のセット合計が改修前後で不変（確定 elapsedSec を集計） |

## 3. 移行検証シナリオ（マイグレーションある時）
| シナリオ ID | 前提 | 操作 | 期待結果 |
|---|---|---|---|
| M1 | periods 欠落の legacy 進行中セッションが IndexedDB にある | アプリ再起動 → 復帰 | `[{startedAt, endedAt}]` を合成して復帰し、保存済み elapsedSec を信頼（履歴の合計が不変） |

## 4. 環境要件差分
| 項目 | 前回 | 今回 | 理由 |
|---|---|---|---|
| 時刻制御 | now 注入 | 中断・再開を時間経過込みで挟み、periods 開閉と経過除外を確認 | period 区間方式の検証 |

## 5. 期待 KPI
| 指標 | 目標 |
|---|---|
| 中断除外の正確性 | 経過 = 計時区間（periods）合計に一致。中断区間は 0 算入 |
| 履歴合計の不変性 | 改修前後で既存セッションの振り返り合計が不変 |
