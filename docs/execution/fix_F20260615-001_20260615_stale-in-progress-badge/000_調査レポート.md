# バグ調査レポート: 計時停止後も「進行中」表示が残る

> **issue / slug**: F20260615-001 / stale-in-progress-badge
> **重大度**: medium
> **再現可否**: 可（コード解析で経路特定済、実機 console.log 確認を推奨）
> **報告日**: 2026-06-15
> **対象**: ../README.md（execution）

---

## 1. 症状
計時を正規に終了（「セット終了」or 自動終了）したのに、セット一覧の当該セット名に「・進行中」バッジが残る／別セットの活動画面（`/run`）で「『X』が計時中です」の復帰導線が出続ける。

## 2. 期待挙動
計時が done になったら、一覧の「進行中」バッジと `/run` の「計時中です」復帰導線は即座に消える（進行中セッションは高々 1 つ・done は進行中でない、R20260614-001 の不変条件）。

## 3. 再現手順
1. セットを作り活動を開始（計時 running）
2. 「セット終了」で正規終了（status=done）
3. セット一覧（`/sets`）へ戻る／別セットの `/run` を開く
4. → 当該セットに「・進行中」バッジ／「計時中です」が残って見える（特に同一ナビゲーション内で observer が remount されない場合）

| 備考 | react-query キャッシュ（gcTime 5分）に依存するタイミング系のため、実機で console.log 確認推奨（CLAUDE.md UI タイミング debug 方針） |

## 5. 影響範囲
- 影響機能: セット一覧の進行中バッジ（SetListPage）/ `/run` の別セット計時中ガード（RunInner）
- データへの影響: なし（計時自体は正しく done 化されている。表示・ガードのみ stale）
- 機能影響: stale ガードが残ると別セットの新規開始が一時的にブロックされ得る（medium）

## 6. 暫定回避策
ページを手動リロード（full mount で `["in-progress-session"]` が refetch され解消）。

- 起点: ユーザー指定（原メッセージの埋め込み `/flow:fix`）。

## 10. タイムラインから抽出した仮説
- 仮説1（有力）: `["in-progress-session"]` query が**セッション done 化時に invalidate されない**。`exec.end()` は IndexedDB を done に更新するが react-query キャッシュは更新されず、observer が remount されない経路で stale 値（進行中）を表示し続ける。
- 仮説2: ghost session（過去の running/paused 残存）を findInProgress が拾う。R20260614-001 で可視化対象だが、done 化漏れがあれば該当。→ 主因は仮説1（invalidation 欠落）と判断。
