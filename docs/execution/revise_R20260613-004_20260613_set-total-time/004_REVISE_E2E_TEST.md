## 1. 変更 UC シナリオ

### UC: 計時画面でセット全体の経過時間を見る
| シナリオ ID | 前提 | 操作ステップ | 期待結果 |
|---|---|---|---|
| E1 | 複数アイテムのセット | 開始 → しばらく計時 | `set-elapsed`（セット合計）が表示され、現在アイテムの経過と区別できる |
| E2 | アイテム 1 を終了 → 次のアイテムへ | 次の活動へ進む | セット合計は前アイテムの確定分を含んだまま増え続ける（リセットされない） |
| E3 | 一時停止 | 一時停止する | セット合計の増加が止まる（pause 中は凍結）。再開で再び増える |
| E4 | セット終了 | セット終了する | 完了メッセージが出る（計時中の合計表示は消える） |

## 2. リグレッションシナリオ（既存 UC、重要度高）
| UC | シナリオ ID | 確認観点 |
|---|---|---|
| 計時の基本 | R1 | 進行中アイテムの経過（mm:ss）・開始/現在時刻が従来どおり |
| 永続/復帰（R20260611-001） | R2 | リロード/復帰後もセット合計が正しく復元される |
| 達成記録 | R3 | 実行確定で達成日が記録される（合計表示は記録に影響しない） |
| 振り返り総覧（R20260613-001） | R4 | 振り返り側のセット合計表示は従来どおり |
