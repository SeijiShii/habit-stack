> **最終更新**: 2026-06-14

## 1. 変更 UC シナリオ

### UC-EX-START: セット詳細から「開始」一発で計時画面へ
| シナリオ ID | 前提 | 操作ステップ | 期待結果 |
|---|---|---|---|
| E1 | アイテム 1 件以上のセット set-A、進行中なし | セット一覧 → set-A → 詳細「開始」を押す | `/run/set-A` へ遷移し、中間「開始」ボタンを介さず計時中セクション（現在アイテム・経過）が表示される（auto-start） |
| E2 | E1 直後 | 計時中セクションを確認 | 一時停止／セット終了／メモ等が操作でき、二段構えの中間ページが存在しない |

### UC-EX-RESUME: 計時中に一覧へ移動 → 進行中 → 復帰
| シナリオ ID | 前提 | 操作ステップ | 期待結果 |
|---|---|---|---|
| E3 | set-A を計時中 | ナビでセット一覧 `/sets` へ移動 | set-A 行に「進行中」バッジが表示される |
| E4 | E3 の状態 | 一覧で進行中の set-A を選択 | `/run/set-A`（活動画面）へ戻り、計時が継続している（経過がリセットされない・破棄されない） |
| E5 | E3 の状態 | 一覧で進行中でない set-B を選択 | `/sets/set-B`（詳細）へ遷移（活動画面ではない） |

### UC-EX-DOUBLE: 別セット進行中での二重開始禁止
| シナリオ ID | 前提 | 操作ステップ | 期待結果 |
|---|---|---|---|
| E6 | set-A を計時中 | URL/導線で `/run/set-B` を開く | set-B が二重開始されず、`『set-A』が進行中です` + set-A への復帰リンクが表示される |
| E7 | E6 の状態 | 復帰リンクを押す | `/run/set-A` へ戻り計時継続。set-B の新規 session は作られていない |

## 2. リグレッションシナリオ（既存 UC、重要度高）
| UC | シナリオ ID | 確認観点 |
|---|---|---|
| 計時の基本 | R1 | 現在アイテム経過（mm:ss）・開始/現在時刻・セット合計（R20260613-004）が従来どおり |
| 永続/復帰（R20260611-001） | R2 | リロード後も進行中が復元され、一覧バッジ + `/run` 復帰が成立する |
| 自動終了 | R3 | 放置による autoEnd（`decideRecovery`）の挙動が不変 |
| ログイン時終了（UC-EX-LOGIN-END） | R4 | `/account` 遷移で進行中が終了する挙動が不変（バッジは終了後消える） |
| 達成記録 | R5 | 1 アイテム以上実行で達成日が記録される（導線変更は記録に影響しない） |

## 3. 移行検証シナリオ（マイグレーションある時）
なし（データ移行・スキーマ変更なし）。既存端末に残る進行中セッションは、本改修後に一覧バッジ + 復帰導線で回収可能になることを E3/E4 でカバー。

## 4. 環境要件差分
| 項目 | 前回 | 今回 | 理由 |
|---|---|---|---|
| ルーティング | 単一ページ検証中心 | `/sets` ⇄ `/run/:id` ⇄ `/sets/:id` の遷移を跨いで検証 | 復帰導線・遷移分岐の確認 |
| 進行中状態の事前投入 | — | set-A を進行中にした上で別画面/別セットを操作 | 可視化・二重開始禁止の検証 |

| 指標 | 目標 |

| 計時中セッションの可視性 | 計時中はどの画面からも 1 タップで活動画面に復帰できる |
