# クレーム調査レポート

**claim id**: C20260610-001
**実施日**: 2026-06-10
**対象機能**: execution
**緊急度推定**: medium

## 1. クレーム原文

```
活動開始ボタンを押したとき、経過時間が00:00のままで進まない
終了して記録された時間を見ると時間が進んではいるようだ
経過時間だけでは分かりづらいので、計時中のかつどうについては開始時刻と現在時刻、経過時間を表示する
```

## 2. 分解結果

### 2.1 期待挙動 (Expected)
- (A) 活動を開始したら、画面の「経過時間」が刻々と進む（リアルタイムにカウントアップする）。
- (B) 計時中（running）のアイテムについては、経過時間だけでなく **開始時刻** と **現在時刻** も併せて表示してほしい（経過時間だけでは分かりづらいため）。

### 2.2 現実挙動 (Actual)
- 開始ボタンを押しても「経過時間」表示が `00:00` のまま固まり、進まない。
- 「終了」して記録された時間を見ると、実際には時間は経過している（記録値は正しい）。
- → 記録（タイムスタンプ差分）は正しいが、**計時中のライブ表示が更新されない**。
- 開始時刻・現在時刻は表示されていない（経過時間のみ表示）。

### 2.3 発生条件
- 操作手順: 活動セットの「開始」→ running 状態のまま経過時間表示を観察。
- 環境: アプリ全般（特定環境依存ではなく実装上の常時再現）。
- 時刻: 計時中（status=running / paused）の間、常時。
- ユーザー属性: 全ユーザー。

### 2.4 影響範囲
- 該当ユーザー: 全ユーザー（execution を使う全員）。
- 業務影響: 実行中に経過がわからず「ちゃんと計れているのか」不安になる UX 劣化。記録自体は正しいためデータ破損はなし。
- データ影響: なし（記録値は正確）。

### 2.5 報告経路
- 経路: 直接（オーナー seiji による動作確認フィードバック）。
- 温度感: 冷静（事象＋改善要望を併記）。

### 2.6 報告者文脈
実行中に「今どれくらい経ったか」をその場で把握しながら活動したい。固まった `00:00` では計れているか不安なので、開始時刻・現在時刻・経過時間を揃えて見たい。
