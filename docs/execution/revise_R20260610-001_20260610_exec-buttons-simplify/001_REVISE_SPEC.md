## 1. 変更概要

実行中（running）画面の操作ボタンを簡素化する。アイテム単位の「終了」ボタンは意味が曖昧（押しても status は running のままで、アイテムが「終了済み」になるだけ・ボタン群が再表示され何が起きたか分かりにくい）なため削除する。「一時停止」と「次の活動へ」で計時運用は必要十分。あわせて「次へ」（running）/「次を開始」（paused）の表記ゆれを「次の活動へ」に統一する。**表示層（ExecutionPage）のみの変更**で、状態機械・記録ロジックは不変。

## 2. 変更前 vs 変更後

### 2.1 UC 変更
| UC ID | 変更前 | 変更後 | 理由 |
|---|---|---|---|
| UC4 (running 操作) | ボタン: 終了 / 次へ(非最終) / 一時停止 / セット終了 | ボタン: **次の活動へ**(非最終) / 一時停止 / セット終了（**終了を削除**） | 「終了」は遷移を伴わず状態が曖昧。一時停止 + 次の活動へ + セット終了で十分 |
| UC4 (paused 操作) | ボタン: 同じ活動を再開 / 次を開始 | ボタン: 同じ活動を再開 / **次の活動へ** | 「次を開始」を「次の活動へ」に統一 |

### 2.2 入出力変更（画面）
| 対象 | 変更前 | 変更後 | 互換性 |
|---|---|---|---|
| running「終了」ボタン | あり（onClick=endItem） | **削除** | UI のみ。endItem(状態機械の endCurrentItem) は nextItem/endSession 経由で内部的に呼ばれ続けるため挙動互換 |
| running 次ボタン文言 | 「次へ」 | 「次の活動へ」 | 文言のみ |
| paused 次ボタン文言 | 「次を開始」 | 「次の活動へ」 | 文言のみ |

## 4. 後方互換性

- **互換維持**: ✅
- データ/API/スキーマ非変更。アイテムの終了確定は「次の活動へ」（nextItem→endCurrentItem）/「セット終了」（endSession→endCurrentItem）で従来どおり行われ、達成記録（daily_achievements upsert）も不変。
- 「終了」ボタンに依存していた操作系の唯一の効果（アイテムを終了済みにする）は、次アイテムへ進む/セット終了する操作に内包されるため機能喪失なし。

## 7. 詳細仕様（新仕様）

### 7.1 詳細 UC（新仕様）
UC4 実行フローのうち**操作 UI のみ**変更:
- running 中の操作ボタン: **「次の活動へ」（最終アイテムでは非表示）/ 「一時停止」/ 「セット終了」**。
  - 「次の活動へ」: nextItem（現アイテムを終了確定 → 次 record 開始。最終なら endSession で done）。
  - 「一時停止」: pause（status=paused）。
  - 「セット終了」: endSession（現アイテム終了確定 + done + 達成 upsert）。
- paused 中の操作ボタン: **「同じ活動を再開」/ 「次の活動へ」**。
  - 「次の活動へ」: nextItem。
- 最終アイテムの running では「次の活動へ」を非表示（従来の isLast 挙動を踏襲）→ 完了は「セット終了」。

### 7.2 入出力（新仕様）
§2.1/§2.2 のとおり。状態遷移 API（startSession/endItem/pause/resumeSame/nextItem/endSession）は不変。`endItem`（状態機械 endCurrentItem）は UI から直接呼ぶボタンが無くなるが、API/フックメソッドとしては残置（nextItem/endSession が内部利用）。
