## 1. 変更概要

グローバルヘッダのアプリタイトル「つみあげルーティン」（兼ホームリンク）が、狭幅でナビ項目が詰まりエリプシス（…）になる。タイトル左にブランドロゴを追加し、**幅が確保できるときは「ロゴ + タイトル」、幅が足りないときはロゴのみ**に縮退させる。ロゴは既存 `public/favicon.svg`（積み上がるブロック + amber の達成点）と同じモチーフをインライン SVG 化して用いる。表示のみの変更で、ルーティング・データ・契約に影響しない。

## 7. 詳細仕様（新仕様）

### 7.1 詳細 UC（新仕様）
- ホームリンクは常にロゴを表示。
- 利用可能幅が十分: ロゴ + タイトルテキストを横並び表示。
- 利用可能幅が不足: タイトルテキストを `display:none` で隠し、ロゴのみ表示（エリプシス発生を回避）。
- 縮退の閾値は CSS で制御（コンテナ/ビューポート幅ベース）。JS 計測は使わない（SSR/初期描画でちらつかせない）。

### 7.2 入出力（新仕様）
- `BrandLogo` props: `{ size?: number }`（既定 20）。装飾要素のため `aria-hidden="true"`。
- ホームリンク: `aria-label="つみあげルーティン（ホーム）"`。

### 7.3 データモデル（新仕様）
なし。

### 7.4 バリデーション・エラー（新仕様）
なし。

### 7.5 機能固有 NFR + 既存連携（新仕様）
- レイアウトシフト最小（ロゴは固定サイズ、`width`/`height` 明示）。
- 既存の入口リード（O41）・フッタ法務（O55）・フィードバック導線（O40）は不変。
- design-system トークン（`--color-primary` #3F7A6E / `--color-accent` #E8A23D）を SVG 配色に使用。

## 9. 未決事項

### [論点-001] 縮退の閾値手法（解決済み・auto-pick）
- **詰めるべき問い**: 縮退を media query（ビューポート幅）と container query（ヘッダ幅）のどちらで判定するか。
- **推奨 / 決定**: CSS の container query を優先し、未対応環境向けに max-width media query をフォールバックとして併用。理由: ヘッダ幅はサイドバー有無等で変わり得るため、要素幅基準のほうが堅牢。実装で困難なら media query 単独に縮退可（許容）。
- **判断**: auto-pick で確定（Class A、可逆）。
