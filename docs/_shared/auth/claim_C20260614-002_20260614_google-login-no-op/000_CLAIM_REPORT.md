## 2. 分解結果

### 2.1 期待挙動 (Expected)
アカウント画面の「Google で引き継ぐ」ボタンを押すと、Google の OAuth 同意フロー（リダイレクト）が始まり、ローカルデータを引き継いだ authed owner にアップグレードできる。

ボタンは表示されている（= クライアント Clerk 初期化済 / 「設定済み」）が、押しても**何も起きない**（リダイレクトが始まらない、エラー表示もない）。

### 2.3 発生条件
- 操作: 本番 https://habit-stack.givers.work の /account で「Google で引き継ぐ」を押下
- 環境: 本番（production Clerk instance = `pk_live_` / `sk_live_`）
- 範囲: 全ユーザー（特定アカウント依存ではない）

### 2.4 影響範囲
- 該当ユーザー: Google 引き継ぎ/複数端末同期/課金を試みる全ユーザー
- 業務影響: O22 段階的認証が本番で機能不全 → データ引き継ぎ・課金導線がブロック
- データ影響: なし（ローカルデータは保持、引き継ぎが始まらないだけ）

### 2.5 報告経路
