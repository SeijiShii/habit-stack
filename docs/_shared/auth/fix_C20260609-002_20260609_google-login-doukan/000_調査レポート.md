## 1. 症状
本番アプリ（https://habit-stack.givers.work）に **ログイン/アカウント連携の動線が一切ない**。匿名ゲストとして使い始めることはできるが、Google アカウントに連携してデータを引き継ぐ・複数端末で同期する手段が UI 上に存在しない。

## 2. 期待挙動
concept §1.1「引き継ぎたいときだけ Google ログイン」/ §1.2「デバイス間同期・課金のタイミングで Google ログインを段階的に要求し、ローカルデータを引き継ぐ」/ auth SPEC §3 `linkWithGoogle()`「匿名ユーザーを Google OAuth でアップグレード（同一 owner id を維持 or 移行処理を起動）」。= データを引き継ぎたいユーザーが UI から Google 連携できるべき。

## 3. 再現手順
1. https://habit-stack.givers.work を開く（匿名ゲストで起動）
2. ヘッダー / 任意の画面でログイン・アカウント連携の導線を探す
3. → どこにも存在しない（ヘッダーは「つみあげルーティン / セット / 継続」のみ）

## 4. 再現環境
| 項目 | 値 |
|---|---|
| ブラウザ | 全ブラウザ |
| OS | 全 OS |
| アプリバージョン | 本番 HEAD（commit 5272fe8 時点で確認） |
| データ条件 | 任意（匿名ゲスト） |

## 5. 影響範囲
- 影響ユーザー: 端末変更・複数端末・データ保全を望む全ユーザー
- 影響機能: 中核価値「匿名 → アカウント連携でデータを失わない」(UC8) が機能不全
- データへの影響: あり（端末紛失・ストレージ消去でローカルデータが復旧不能。引き継ぎ手段が無い）
- セキュリティ影響: なし（owner-check / guest セッションは正常稼働）

## 6. 暫定回避策
なし（同一端末・同一ブラウザで使い続ける限りは guest データ継続。引き継ぎは不可）。
