# _shared/auth E2E テスト計画（アカウント切替時の計時停止条件緩和 + 同期ポリシー）

> **入力**: `./001_REVISE_SPEC.md`, `../../../concept.md` §1.1 UC, 既存 `../../../execution/004_execution_E2E_TEST.md`
> **最終更新**: 2026-06-15
> **注**: Google ログイン/サインアウトの実 OAuth 経路は実 Clerk キーが要るため、headless E2E では Clerk を keyless/stub にした「アカウント切替シミュレーション」+ ローカル wipe / 進行中保存の観測で代替する。実キー経路は `/flow:release` のローカル実機確認で担保。

---

## 1. 変更 UC シナリオ

### UC-ACCT-SWITCH-CONFIRM: 切替時の確認付き停止
| シナリオ ID | 前提 | 操作ステップ | 期待結果 |
|---|---|---|---|
| E-01 | セット作成 → 活動を開始（計時 running） | `/account` へ遷移 → 何もせず `/sets` へ戻る | 計時は継続（停止しない）。一覧に「進行中」バッジが残る |
| E-02 | 計時 running 中 | `/account` で「Google でログイン」押下 | 「計時中の活動を停止しますか」確認が出る |
| E-03 | E-02 で「OK（停止して続行）」 | — | 進行中が保存され done 化（記録が残る）→ ログインフロー開始 |
| E-04 | E-02 で「キャンセル」 | — | 計時継続。ログインは始まらない。一覧に進行中バッジが残る |
| E-05 | 連携済みアカウント + 計時 running | `/account` で「サインアウト」押下 → 確認 OK | 進行中保存 → サインアウト |

### UC-ACCT-SIGNOUT: サインアウトでデバイス削除
| シナリオ ID | 前提 | 操作ステップ | 期待結果 |
|---|---|---|---|
| E-06 | 連携済み、デバイスにデータあり | サインアウト（進行中なし） | デバイスのローカルデータが消える（`/sets` が空）。サーバ（アカウント）データは保持され、再ログインで復元可（stub では「wipeOwner が呼ばれた」で観測） |

### UC-ACCT-OVERWRITE: 既存データ持ちログインで上書き
| シナリオ ID | 前提 | 操作ステップ | 期待結果 |
|---|---|---|---|
| E-07 | デバイスにゲストデータ A、既存アカウントにデータ B | 既存 Google でログイン（fallback 経路） | デバイスがデータ B で上書き（A は消え B が見える）。stub では旧 guest owner の wipe → account データ表示で観測 |

### UC-ACCT-LINK: 未連携ログインでデータ保持・アップロード（要望3/4 の核）
| シナリオ ID | 前提 | 操作ステップ | 期待結果 |
|---|---|---|---|
| E-08 | デバイスにゲストデータあり（未連携） | 未連携 Google で初めてログイン | 作成済みセット・活動が**消えずに残り**、アカウントに帰属（アップロード）。`/sets` に同じデータが見える |

## 2. リグレッションシナリオ（既存 UC、重要度高）

| UC | シナリオ ID | 確認観点 |
|---|---|---|
| R20260614-001 計時中可視化 | E-R1 | 計時 running 中に他画面（一覧/振り返り）へ移動しても進行中バッジ + 復帰導線が機能。`/account` 閲覧でも消えない |
| R20260614-001 二重開始防止 | E-R2 | 別セット進行中に `/run/:other` 直叩きで二重開始させず復帰導線 |
| R20260611-002 O54 セルフ削除 | E-R3 | 「全データを削除」（purgeAllData）はサーバ + ローカル両方削除（signOut のローカルのみ wipe と区別され、両方グリーン） |
| **埋め込み fix（別起票）リグレッション観点** | E-R4 | **計時を正規に終了（「終了」操作 or 自動終了）した後、一覧/復帰導線に「進行中（計時中）」表示が残らない**ことを確認。残る場合は別 `/flow:fix execution` で根本原因（ghost session / react-query キャッシュ未 invalidate）を調査・修正する（本 revise では観点として固定するのみ） |

## 3. 移行検証シナリオ（マイグレーションある時）

- 該当なし（DB スキーマ変更・データ移行なし）。

## 4. 環境要件差分

| 項目 | 前回 | 今回 | 理由 |
|---|---|---|---|
| Clerk | keyless/stub（offline-first 検証） | 継続。アカウント切替は stub で wipe/保存を観測 | 実 OAuth は release 実機確認へ |
| IndexedDB | fake-indexeddb（unit）/ 実ブラウザ（E2E） | 継続。wipeOwner 後の空表示を観測 | デバイス削除の検証 |
