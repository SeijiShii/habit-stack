## 1. 症状
Google ログイン状態で前日夕方に作成・計時・振り返り登録まで完了した活動セットのうち、
「夕方の勉強」セットとその実績（計時・振り返り）データが、翌朝には消失。「朝の勉強」は残存。
= **パーシャルなデータ消失**（全消去ではない）。ログイン状態は継続中。

## 2. 期待挙動
ログイン状態で作成・記録まで完了したデータは、再起動・翌日をまたいでも両方保持される
（concept §1.1/§1.2 local-first 保持、R20260615-001 §UC-ACCT-LINK「データは保持」、
同 §4「連携済みは再 pull で復元可能」）。パーシャル消失は仕様上あってはならない。

## 3. 再現手順
報告者の実機手順は特定困難。ただし**欠陥のロジック再現手順は決定的**:

1. ゲスト状態（owner = G1）でアプリを開き、活動セット A を作成（ローカル IndexedDB + outbox に upsert(A,G1) 積載）。
2. 「Google でログイン」を実行 → 選んだ Google が**既存 Clerk アカウントに連携済み** →
   `createExternalAccount` 失敗 → `AuthProvider.tsx:91-120` の fallback で `markDeviceOverwrite()` →
   `signInWithGoogle()` で OAuth リダイレクト → **既存アカウント U にサインイン（userId が G1→U に churn）**。
3. 復帰後の app 初期化 `repos.ts:49-53` で `consumeDeviceOverwrite()` = true →
   `store.wipeOtherOwners(U)` 実行 → `wipeOwner(G1)`（`localStore.ts:146-171`）が
   **G1 のエンティティ + 未送信 outbox を物理削除** → セット A は同期前に消滅し恒久喪失。
4. 以後 U で作成したデータ（例: セット B）は owner=U なので残存 → **パーシャル消失が成立**。

> 実機での「パーシャル」は、2 セットの作成が userId churn（手順 2 のログイン）を**挟んで**行われ、
> 片方が G1 owner・片方が U owner になったために、wipe が G1 側だけを消したと整合する。

## 5. 影響範囲
- 影響ユーザー数: 確認 1 名（報告者）。**既存アカウントへサインインする全ユーザーに潜在波及**
  （ゲスト中に作成したローカル未同期データを持つ場合）。
- 影響機能: activity-sets / execution / streak-summary（全ローカルデータ）。
- データへの影響: **あり・不可逆の疑い**（ローカル物理削除 + 未送信 outbox も削除 → サーバ未到達なら復元不能）。
- セキュリティ影響: なし。

## 6. 暫定回避策
- ユーザー向け: 既存 Google アカウントへサインインする前に、データがサーバへ同期済みであることを担保する
  手段が現状ない（=有効な回避策なし）。
- 運用向け: **`wipeOtherOwners` 呼び出しの即時無効化**が最小リスクの止血策。R20260615-001 spec-review R2 が
  「物理 wipe しなくても owner スコープ read（`getAllByOwner`）で混在表示は起きない」と明記しており、
  この破壊的処理は**正当性に不要**。→ §002_FIX_PLAN で即時 mitigation として採用。

## §12 タイムラインから抽出した仮説（5 Whys の起点）
- 仮説1（採用・主因）: 既存アカウントサインインで userId が churn → ローカルデータが旧 owner に取り残される →
  `wipeOtherOwners` が物理削除（outbox ごと）→ 同期前データが恒久喪失。
- 仮説2（採用・寄与）: `reassignOwner` がローカル未対応かつ未配線 → owner churn 時のローカルデータ移行が皆無。
- 仮説3（採用・寄与）: `markDeviceOverwrite` が「既存アカウント連携済み」を一律「デバイス上書き」と解釈 →
  保持すべきローカルゲストデータを破棄対象にする。
- 仮説4（降格）: sync conflict による soft-delete。`conflict.ts` は単純 LWW でサーバ tombstone がない限り
  削除しない → 本症状の主因ではない（サーバ側 tombstone の混入が無ければ非該当）。
