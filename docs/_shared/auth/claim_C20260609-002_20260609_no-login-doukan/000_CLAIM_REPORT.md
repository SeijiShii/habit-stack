# クレーム調査レポート

**claim id**: C20260609-002
**実施日**: 2026-06-09
**対象機能**: _shared/auth（+ app-shell UI 配線）
**緊急度推定**: high（本番公開済・契約済み能力の欠落）

## 1. クレーム原文

```
ログインがない
[flow]契約違反。画面や機能を作っても動線がない。auditで検知しない
```

## 2. 分解結果

### 2.1 期待挙動 (Expected)
- データを引き継ぎたい / 複数端末で同期したいとき、**Google ログイン（アカウント連携）の動線**がアプリ上に存在し、そこから段階的に認証してローカルデータを引き継げる。
- 概念が「引き継ぎたいときだけ Google ログイン」を謳う以上、ユーザーから見て「ログインする手段」が UI 上に必ず在るべき。

### 2.2 現実挙動 (Actual)
- アプリには **ログイン動線が一切無い**。`AuthProvider` は匿名ゲスト ticket セッションを自動確立するだけで、`linkWithGoogle` 関数も sign-in ボタンも Google 連携フローも存在しない（grep: 実装ゼロ、コメント参照のみ）。
- `AppLayout` のリード文も「ログインなしで今すぐ始められます」と明記。= 永続的にゲストのみ。

### 2.3 発生条件
- 操作手順: アプリ起動後、どの画面からもログイン/アカウント連携に到達できない。
- 環境: 本番 https://habit-stack.givers.work（Clerk production instance verified、guest ticket は稼働）。
- ユーザー属性: データ引き継ぎ・複数端末同期・課金を行いたいユーザー。

### 2.4 影響範囲
- 該当ユーザー: 端末変更/複数端末/データ保全を望む全ユーザー。
- 業務影響: 概念の中核価値「匿名→アカウント連携でデータを失わない」(§1.2 UC8) が機能不全。
- データ影響: 端末紛失・ストレージ消去でローカルデータが復旧不能（引き継ぎ手段が無い）。

### 2.5 報告経路
- 経路: 開発者（seiji）直接 + [flow] フィードバック。
- 温度感: 構造的指摘（契約違反 + audit 検知漏れの是正要求）。

### 2.6 報告者文脈
- 「画面や機能を作っても動線がない」事故が PJ をまたいで頻発しており、本件（契約済み認証動線の欠落）をその代表例として、PJ 修正と audit 強化の両面を求めている。
