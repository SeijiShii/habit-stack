## 1. 変更 UC シナリオ

### UC-AUTH-DELETE: 全データのセルフサービス削除
| シナリオ ID | 前提 | 操作ステップ | 期待結果 |
|---|---|---|---|
| E-DEL-01 | ゲストでセット + アイテム + 達成記録を作成済み | `/account` → 「全データを削除」→ 確認「削除する」 | 削除後トップへ。セット一覧が空（データ消失）= ゲストのフレッシュ状態 |
| E-DEL-02 | 同上 | 「全データを削除」→ 確認「キャンセル」 | データは残る（削除されない） |
| E-DEL-03 | keyless ゲスト（オフライン相当、実キーなし＝ローカルゲスト） | 削除実行 | ローカルデータが消える（サーバ無しでもエラーなく完結） |

> 注: ローカル headless（実キーなしゲスト owner）= Class A。E-DEL-01/03 はローカル wipe の実ブラウザ検証を担う。
> サーバ DB の物理削除（deleteAllData）は unit（U-DEL-01）で検証（E2E は実 DB を持たないため）。
