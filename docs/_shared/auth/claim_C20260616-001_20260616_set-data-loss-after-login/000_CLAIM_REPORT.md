# クレーム調査レポート

**claim id**: C20260616-001
**実施日**: 2026-06-16
**対象機能**: _shared/auth（同期/owner 境界。実体は _shared/local-sync と協働）
**緊急度推定**: high（ユーザーデータの不可逆消失クラス。再現条件未特定だが影響は致命的）

## 1. クレーム原文

```
昨日夕方時点でGoogleログインした状態で活動セットを作成し、計時し、振り返りに登録される
ところまで確認した。その時点で「朝の勉強」「夕方の勉強」セットが登録されている状態だった。
今朝起きたら「朝の勉強」しかなくなっており「夕方の勉強」セットとその実績データが消えている。
Googleログインしている状態だ
Googleログイン状態が変化するとデバイスに保存されているものが上書きされることもあると
思うが、昨日の夕方と比べてデータがパーシャルに残っているので、完全に上書き削除された
という状況でもなさそう。
再現条件の特定が難しいが検討してください
```

## 2. 分解結果

### 2.1 期待挙動 (Expected)
Google ログイン状態で作成・計時・振り返り登録まで完了した活動セット（「朝の勉強」「夕方の勉強」）は、
セッションをまたいで（翌朝の再起動後も）**両方とも保持されている**こと。ログイン状態が継続している
限り、作成済みデータが勝手に消えない。

### 2.2 現実挙動 (Actual)
翌朝の再起動後、「夕方の勉強」セットとその実績（計時・振り返り）データだけが消失。「朝の勉強」は残存。
= **パーシャルなデータ消失**（全消去ではない）。Google ログイン状態は継続中。

### 2.3 発生条件
- 操作手順: 昨日夕方に Google ログイン状態でセット作成 → 計時 → 振り返り登録（両セットとも完了確認済）。
  翌朝アプリを再度開いた。
- 環境: 本番（昨日 R20260615-001 + F20260615-001 をデプロイ済）。クライアントは PWA/ブラウザ（端末詳細は未取得）。
- 時刻: 喪失は昨日夕方〜今朝の間（夜間/再起動境界）。
- ユーザー属性: Google 連携済み（既存アカウントの可能性が高い＝過去にも利用＝「確認した」発言）。
- **再現条件は未特定**（報告者も特定困難と明言）。

### 2.4 影響範囲
- 該当ユーザー: 少なくとも 1 名（報告者）。同経路を踏む全 Google 連携ユーザーに潜在波及。
- 業務影響: 作成・計時・振り返りの成果が消える = プロダクト中核価値（継続の記録）の信頼性毀損。
- データ影響: **不可逆消失の疑い**（ローカル物理削除 + サーバ未保持なら復元不能）。

### 2.5 報告経路
- 経路: 開発者（seiji）自身のドッグフーディングからの直接報告。
- 温度感: 冷静・分析的（パーシャル消失という観察を添えて再現検討を依頼）。

### 2.6 報告者文脈
報告者が達成したかったこと: 自分の習慣（朝/夕の勉強）を継続記録すること。そのために前日に正しく
セットを作り計時・振り返りまで完了させたのに、翌日その一部が失われており、記録の継続が破綻した。
