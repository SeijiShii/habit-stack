# バグ調査レポート: Google 連携が aged guest session で reverification 403 になり無反応

> **issue / slug**: C20260614-002 / google-link-reverification
> **重大度**: high
> **再現可否**: 確認済（本番実機 + chrome://inspect リモートデバッグ）
> **報告日**: 2026-06-14
> **対象**: ../README.md（_shared/auth）
> **起点クレーム**: ../claim_C20260614-002_20260614_google-login-no-op/001_TRIAGE.md §7・§8（根本原因確定）

---

## 1. 症状
本番 https://habit-stack.givers.work の /account「Google で引き継ぐ」が、**PC（フレッシュなセッション）では機能する**（accounts.google.com に有効な client_id で遷移）が、**スマホ（数十分放置した古いゲストセッション）では押しても何も起きない**（エラーも出ない）。

## 2. 期待挙動
concept §1.1/§1.2「引き継ぎたいときだけ Google ログイン」/ auth SPEC §3 `linkWithGoogle()`「匿名 user を Google OAuth でアップグレード（O22 段階的認証）」。= aged guest を含む全ユーザーが /account から Google 連携を開始でき、データを引き継げるべき。

## 3. 再現手順
1. 本番 /account を開き、ゲストセッションを数十分放置（または前日のセッションを再利用）して **session を aged 化**する。
2. 「Google で引き継ぐ」を押す。
3. → 何も起きない（PC のフレッシュセッション or シークレットウィンドウでは Google 同意画面へ遷移する＝対比で確認）。

## 4. 再現環境
| 項目 | 値 |
|---|---|
| ブラウザ | Android Chrome（通常ブラウザ）で顕在化。PC Chrome では fresh のため非顕在 |
| OS | mobile 固有ではなく **session 鮮度**の問題（aged session を持つ端末で出る） |
| アプリバージョン | 本番 HEAD（fix C20260609-002 デプロイ後） |
| データ条件 | aged guest session（reverification window 切れ） |

## 5. 影響範囲
- 影響ユーザー: aged guest session を持つ全ユーザー（O22 段階的認証＝データ引き継ぎ / 複数端末同期 / 課金の前提が aged guest で恒久不全）。
- 影響機能: 中核価値「匿名 → アカウント連携でデータを失わない」が aged session で機能不全。
- データへの影響: なし（連携が始まらないだけ。データ損失・破壊はない）。
- セキュリティ影響: なし（owner-check / guest セッションは正常稼働）。

## 6. 暫定回避策
ユーザー操作での回復は不能（ゲストにはサインアウト動線が無く `AccountPage` は isLinked 時のみ表示、リロードでも `AuthProvider` が isSignedIn なら ticket 再発行を skip）。= **恒久的に詰む**。fresh セッションを持てる新規端末/シークレットからのみ連携可。

## 7. 関連ログ・スタックトレース
chrome://inspect リモートデバッグで確定:
```
POST https://clerk.habit-stack.givers.work/v1/me/external_accounts → 403 Forbidden
Uncaught (in promise): "You need to provide additional verification to perform this operation"
  at u.createExternalAccount (clerk.browser.js) → linkWithGoogle → onLink
```

## 8. 過去類似バグ
- 前回 fix C20260609-002（動線・コードを実装）。本件は前回 fix が **本番 aged session 経路を検証していなかった**ことで残った欠陥（O22 broad-match mask と同型の「動線もコードも有るのに aged session で動かない」）。
